# GET VS POST 차이 

## GET 과 POST는 HTTP 프로토콜에서 사용되는 메소드.

## GET 

- 서버로부터 데이터를 요청하여 응답을 받는데 사용되며 , 데이터를 URL의 쿼리 파라미터로 전송. 그렇기 떄문에 데이터가 노출 될 수 있음 . 보안에 취약 

- 브라우저 기록에 저장 . 캐싱이 가능. 

- 데이터 길이 제한 . 

## POST 

- 서버로 데이터를 제출하거나 전송. 요청 본문에 데이터를 담아 전송

- 브라우저 기록에 저장 x . 데이터가 요청 본문에 포함되므로 노출 x , 보안 높음

- 캐싱 x , 항상 새로운 요청 

- 데이터 길이 제한 x
